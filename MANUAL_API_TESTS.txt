

-------------------------------------------------
### 9. GESTIÓN (ADMIN)
-------------------------------------------------
**Nota:** Para probar estos endpoints, necesitas un token de un usuario Administrador.
Inicia sesión con un usuario admin (ej: `admin`, `adminpass`) para obtener el token.

#### 9.1. Gestionar Clientes (CRUD)

- **Listar Clientes:**
  - **Método:** `GET`
  - **URL:** `http://127.0.0.1:8000/api/clients/`
  - **Headers:** `Authorization: Bearer <TOKEN_ADMIN>`

- **Crear Cliente:**
  - **Método:** `POST`
  - **URL:** `http://127.0.0.1:8000/api/clients/`
  - **Headers:** `Authorization: Bearer <TOKEN_ADMIN>`, `Content-Type: application/json`
  - **Body (raw, JSON):**
    ```json
    {
        "username": "cliente_desde_admin",
        "password": "password123",
        "email": "cliente_admin@test.com",
        "first_name": "CreadoPor",
        "last_name": "Admin"
    }
    ```

- **Ver Detalle, Actualizar, Eliminar Cliente (ID=2):**
  - **Ver:** `GET http://127.0.0.1:8000/api/clients/2/`
  - **Actualizar:** `PUT http://127.0.0.1:8000/api/clients/2/` (enviar body con los campos a cambiar)
  - **Eliminar:** `DELETE http://127.0.0.1:8000/api/clients/2/`
  - **Headers:** `Authorization: Bearer <TOKEN_ADMIN>`
